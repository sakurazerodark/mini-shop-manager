# 店小易 (MiniShop) - 极简门店商品管理与收银系统

> 专为一人店、夫妻店、小型便利店设计的桌面端轻量级商品管理和收银系统。
> 数据完全本地化，无需复杂的服务器部署，一键启动，开箱即用。

## 📖 项目简介

**店小易 (MiniShop)** 致力于为小型零售实体店提供最低成本的数字化解决方案。它采用前沿的 Web 技术与桌面应用框架（Electron）结合，将前端展示、后端业务逻辑与 SQLite 数据库打包在一起。它支持从最基础的扫码收银、库存管理、利润统计，到扫码聚合支付（支付宝/微信等）的全流程业务。

## 🛠 技术栈

- **前端框架**: Vue 3 + Vite
- **后端服务**: Node.js + Express.js
- **本地数据库**: SQLite3
- **桌面端构建**: Electron
- **UI 设计**: 纯手工精美原生 CSS，响应式设计布局

## ✨ 功能特性

### 1. 🛒 智能收银台 (POS)
- 支持**扫码枪快速录入**商品条码。
- 支持无条码的**称重商品**或散装商品手动输入计价。
- 购物车灵活管理（修改数量、单品改价、删除）。
- 支持**多支付方式组合**（现金、个人主扫收款码、支付宝当面付、微信支付、云闪付）。

### 2. 📦 商品与库存管理
- **商品档案库**：商品录入、分类、定价（进货价/零售价）、单位管理。
- **智能查码**：集成第三方商品条码 API，扫码自动带出商品名称和信息。
- **库存变动日志**：进货入库、销售扣减、报损出库、盘点校准，每一步变动皆可追溯。
- **报表筛选**：支持按日期区间精准筛选库存出入记录。

### 3. 📊 销售与利润报表
- **营业大盘统计**：实时统计今日/本月营业额、总毛利、订单数、客单价。
- **订单明细流水**：查看历史每一笔订单的商品明细、支付方式及确认来源。
- 订单退款与异常订单处理。

### 4. ⚙️ 系统设置与扩展
- **聚合支付配置**：内置支付宝当面付（沙箱/生产环境）、微信支付 V3、云闪付的密钥配置面板。
- **基础设置**：店铺名称、小票打印配置、API 接口配置。

## 🚀 快速开始

### 1. 环境要求
- [Node.js](https://nodejs.org/) (建议 v18 或以上版本)
- npm、yarn 或 pnpm

### 2. 克隆与安装依赖

```bash
git clone https://github.com/sakurazerodark/mini-shop-manager.git
cd mini-shop-manager
npm install
```

### 3. 运行开发模式

#### 方式一：分别启动前端与后端（推荐 Web 调试）
```bash
# 终端 1：启动 Express 后端服务 (默认端口 8080)
node main/start-backend.js

# 终端 2：启动 Vite 前端服务 (默认端口 5173)
npm run dev
```

#### 方式二：启动 Electron 桌面应用
```bash
# 自动启动前端 Vite 与 Node 后端，并唤起 Electron 窗口
npm run start
```

### 4. 构建生产版本 (打包为桌面 .exe / .dmg)

```bash
npm run build:electron
```
打包成功后，安装包将生成在 `release` 目录下。

## 配置说明

### 支付宝配置

1. 登录支付宝开放平台
2. 进入沙箱环境获取测试账号
3. 在系统设置中填写：
   - 应用 ID (APP ID)
   - 支付宝网关
   - 商户私钥
   - 支付宝公钥

### 微信支付配置

1. 登录微信支付商户平台
2. 获取商户号、API 证书等
3. 在系统设置中填写相关配置

### 商品码 API 配置

支持探数 API 等第三方商品条码查询服务，配置 API Key 后可自动获取商品信息。

## 注意事项

- 当前版本仅支付宝沙箱环境已跑通，生产环境使用前请仔细测试
- 微信和云闪付支付功能尚未完成联调，请勿直接用于生产
- 商品码 API 需要自行申请并配置
- 建议定期备份 `data/shop.db` 数据库文件
